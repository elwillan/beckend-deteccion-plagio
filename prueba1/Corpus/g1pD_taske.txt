La programación dinámica es un método más rápido para resolver problemas que hace uso de una subestructura óptima, subproblemas y memorización superpuestos. No tiene relación con la programación de computadoras; en cambio, es un proceso de encontrar un algoritmo satisfactorio.

La subestructura óptima es el proceso de usar las soluciones opcionales para los subproblemas para encontrar la solución óptima al problema general. Cuando se pueden utilizar las mismas soluciones de subproblemas para resolver varios problemas más grandes, se dice que tienen problemas de superposición superpuestos. La memorización se utiliza para ahorrar tiempo, las soluciones se almacenan en lugar de ser recalculadas. Una solución puede eliminarse una vez que estamos seguros de que ya no será necesaria, en algunos casos se puede calcular de antemano una solución a un problema futuro.

Hay dos enfoques principales para la programación dinámica. El primero es el enfoque de abajo hacia arriba. Aunque no siempre es fácil encontrarlos todos, los subproblemas necesarios se resuelven por adelantado y luego se utilizan para crear soluciones a problemas más grandes. El otro método es el enfoque de arriba hacia abajo, que combina la memorización y la recursividad. El problema principal se divide en subproblemas que se resuelven y almacenan para su uso futuro.
